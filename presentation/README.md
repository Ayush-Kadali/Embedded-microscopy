# Presentation Materials

## Contents

- `slides.pdf` or `slides.pptx` - Main presentation deck
- `demo_script.md` - Demo walkthrough and talking points
- `screenshots/` - UI screenshots (not in git, too large)
- `video/` - Recorded demos (not in git, share via Drive)

## Setup

Person 5 (Presentation lead) will create these files during development.

## For Person 5 (Presentation Lead)

See `docs/MODULE_DEVELOPMENT.md` section "Person 5: Presentation" for:
- Slide deck structure (10-15 slides)
- Demo script template
- What screenshots to capture
- How to record backup video
- Q&A preparation tips

## Timeline

**Day 1-2**: Skeleton slides, screenshots as system develops
**Day 3-4**: Complete slides, practice demo, record video
**Day 5**: Final polish, team practice, ready for judging

## Suggested Slide Structure

1. Title
2. Problem Statement
3. Solution Overview
4. System Architecture
5. Demo (live or video)
6. Technical Details
7. Results & Metrics
8. Impact & Applications
9. Future Work
10. Team & Acknowledgments
11. Q&A

## Tips

- Keep slides visual, minimal text
- Use screenshots of actual system
- Have backup video in case demo fails
- Practice timing (10-15 minutes total)
- Prepare for technical questions
